Enunciado:
Crea un proyecto en Flask que simule un miniblog, donde los usuarios puedan compartir
sus posteos.
Deberás implementar las siguientes funcionalidades:

Modelos:
a) Usuario: Representa a los usuarios registrados en el miniblog. Debe tener campos
como nombre de usuario, correo electrónico y contraseña.
b) Entrada o Post: Representa una publicación realizada por un usuario. Debe contener
campos como título, contenido, fecha de creación y el autor (relacionado con el modelo
Usuario).
c) Comentario: Representa los comentarios realizados en una entrada. Debe tener
campos como el texto del comentario, fecha de creación y el autor (relacionado con el modelo
Usuario).
d) Categoría: Representa las etiquetas asociadas a las entradas. Debe tener un campo
para el nombre de la etiqueta.

Migraciones:
Asegúrate de generar las migraciones necesarias para crear las tablas correspondientes en
la base de datos.

Vistas y templates:
Implementa los templates necesarios para que los usuarios puedan ver, crear posteos
y comentarios (Editar y Eliminar es un plus).

Context processor:
Utiliza un context_processor para proporcionar la lista de categorías disponibles en

Consideraciones:
- Se entrega un repositorio de Github
- Maximo por grupo 3 personas
- Cada uno debe entregar el repositorio en el que trabajo en classroom ya que alli figura la entrega o no de la EFI
- La base de datos debe estár en MySQL
- Fecha de entrega 12/8
